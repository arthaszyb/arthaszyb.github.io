错误的一次排查
2017
年
6
月
13
日
16:39
配置一个
nginx
反向代理到
quan.qq.com
原
nginx
代理配置
##################

upstream quan{

server 183.61.38.230:80;

keepalive 64;

}

upstream quan_https{

server 183.61.38.230:443;

keepalive 64;

}

server {

listen

80;

server_name

quan.qq.com;

#charset koi8-r;

#access_log

logs/host.access.log

main;

location / {

proxy_pass
http://quan/
;

proxy_http_version 1.1;

proxy_set_header Connection "";

}

error_page

500 502 503 504

/50x.html;

location = /50x.html {

root

html;

}

}
#########################

结果访问
400
。
tcpdump
后将文件用
wireshark
读取，查看
http
包，

追踪
tcp
流：

发现
host
是错误的，直接用了
upstream
的名字，于是修改
upstream
的名字为
quan.qq.com
，解决！
其实这里是需要修改
proxy_set_header Connection ""
为
proxy_set_header host $host
即可了。具体请搜索这个参数的意思。
